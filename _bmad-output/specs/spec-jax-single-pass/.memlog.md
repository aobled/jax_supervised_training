- (decision) Why: dual driver — (a) pain: box-fusion bug on close-formation aircraft, topologically inherent to segmentation+cv2.findContours (semantic seg != instance seg), not a tuning issue; (b) opportunity: eliminate the python/cv2 intermediate crop step between UNet detection and CNN classification, run inference as one JAX-native graph. Source: notes-jax-single-pass.md, contexte/motivation.
- (capability) CAP-1 JAX_DETECTOR training: intent=train a new anchor-free point-detection model (heatmap of centers + size regression) replacing UNet segmentation as the detection method, on 224x224 .npz chunks only, no full-HD dataset ever required (ratio ~41x would be prohibitive, confirmed via existing dataset-prep pattern). success=JAX_DETECTOR trains end-to-end via fighterjet_detection_dataset_tools_v2.py + new dedicated task_strategies.py/data_management.py classes and produces usable heatmap+size predictions on held-out validation images. Source: spine AD-9, AD-14, AD-17, AD-18; notes decisions 2026-07-14/15.
- (capability) CAP-2 Unified JAX-native inference (JAX Single-Pass): intent=given a 1920x1080 grayscale frame, a single JIT-compiled function (build_single_pass_predict_fn) produces up to 20 detections with box+class+scores, without cv2.findContours or any python-level crop loop on the hot path. success=build_single_pass_predict_fn runs end-to-end on a real frame, returns {boxes, classes, class_scores, detection_scores, valid_mask} (fixed 20 slots, zero-padded, valid_mask sole authority), with zero cv2 calls on that path. Source: spine AD-9, AD-11, AD-12, AD-13, AD-15, AD-16; notes mermaid diagram.
- (capability) CAP-3 Box-fusion structurally addressed: intent=two aircraft in close/touching formation, which fuse into one box under the current segmentation+findContours pipeline, are predicted as independent per-instance detections under the new point-detection head. success=on a validation set including close-formation cases, the new pipeline yields distinct per-instance detections structurally (point-per-object, not blob-extraction) — known accepted limitation: real overlap (pixel-touching) accuracy still bounded by sparse training signal (~1.36% of images have 2+ boxes, only a handful truly overlapping), not a full fix, see Non-goals. Source: notes 'Découverte clé'/'Convergence des deux sujets'; spine AD-9, AD-19.
- (capability) CAP-4 Classification reused frozen: intent=the existing FIGHTERJET_CLASSIFICATION checkpoint is loaded frozen inside the new inference graph and reused as-is, no retraining. success=FIGHTERJET_CLASSIFICATION weights load read-only into build_single_pass_predict_fn and produce class predictions matching today's separate classification step within validated crop pixel-parity tolerance (see Open Questions). Source: spine AD-14, AD-16, AD-20; notes 'Objectif initial'.
- (constraint) FIGHTERJET_CLASSIFICATION must never be retrained by this work — frozen load only. Source: spine AD-14, AD-16.
- (constraint) No training dataset may require full-HD images; JAX_DETECTOR trains only on 224x224 (or its configured resolution) chunks, deterministic resize happens only at inference composition time. Source: spine AD-12, AD-14; notes 'Ressources' 41x ratio confirmation.
- (constraint) Non-regression / rollback: the entire old pipeline (FIGHTERJET_DETECTION config, AircraftDetectorUNet, DetectionStrategy, DetectionDataset, decode_segmentation_and_detect(_batch), fighterjet_detection_dataset_tools.py, and all their consumers incl. tools/audit_dataset_detection.py and tools/boxes_process_manual_tkinter.py) must remain fully functional end-to-end (training AND inference), unmodified, for the whole epic and after — explicit user requirement, rollback safety net if JAX Single-Pass proves to be a mistake. Source: spine AD-20 (added post-finalize at user's explicit request).
- (constraint) Inference-time composition must be zero-python/cv2 on its hot path (no cv2.findContours, no python crop loop) — the core motivating constraint, not negotiable down to 'mostly JAX'. Source: notes 'Objectif initial'; spine AD-9, AD-11.
- (constraint) Output of build_single_pass_predict_fn is always a fixed 20-slot structure (boxes/classes/class_scores/detection_scores/valid_mask), invalid slots zero-padded; valid_mask is the sole authority for real-vs-empty, never inferred from classification output. Source: spine AD-15.
- (decision) Non-goals: retraining/modifying FIGHTERJET_CLASSIFICATION; backbone+FPN upgrade (spine AD-10, deferred, no measured need); grid-based/anchor-based (YOLO-style) detection (spine AD-19, deferred pending an overlap-specific synthetic data strategy); full/guaranteed resolution of pixel-touching aircraft overlap (known accepted limitation, structural improvement only per CAP-3, not a guarantee); migrating tools/audit_dataset_detection.py and tools/boxes_process_manual_tkinter.py to the new pipeline (optional future work); any CI/CD, hosted service, or deployment/environment change (local + Colab execution unchanged, spine Deferred).
- (decision) Success signal: JAX Single-Pass replaces the manual python/cv2 orchestration in bounding_boxes_with_classification_from_video_generation.py and tools/bounding_boxes_with_classification_from_images_generation.py — a frame flows through one JIT-compiled function from raw pixels to {boxes, classes, scores}, no cv2.findContours call remains on that path, and on a close-formation validation case the new detector yields distinct per-instance detections where the old pipeline fused them into one box — all while FIGHTERJET_DETECTION's old pipeline still runs unmodified as a working fallback (AD-20).
- (question) Does the JAX RESIZE method (1920x1080->224x224, inference) match PIL/LANCZOS (used when preparing JAX_DETECTOR's training chunks) closely enough that the detector doesn't see a distribution shift at inference? Needs a numeric parity test before training, not just visual comparison. Source: spine Deferred; notes 'Risques identifiés'.
- (question) Does jax.scipy.ndimage.map_coordinates crop match cv2.resize's pixel-alignment convention (border vs. center) closely enough that FIGHTERJET_CLASSIFICATION (trained on cv2 crops) doesn't silently regress on JAX crops? Needs a numeric parity test on real images before relying on it. Source: spine AD-11, Deferred; notes 'Risques identifiés'.
- (question) Should JAX_DETECTOR's encoder be initialized randomly or from AircraftDetectorUNet's existing weights (transfer learning)? Could cut training time/cost but no guarantee of compatibility (current encoder shaped for dense segmentation output, not a point heatmap+size head) — to test empirically once training starts, not before. Source: spine Deferred item #9; notes open decision #9.
- (question) Exact values left to story-level, not yet decided: detector output stride/resolution (single source of truth required between peak-extraction and RESCALE, spine AD-13), detection-score threshold and where it lives (spine AD-15), and the heatmap+size .npz interchange schema — key names/shapes/units/gaussian-radius formula (spine AD-18).
- (question) jax.scipy.ndimage.map_coordinates roadmap risk: an unlanded JAX JEP proposes moving it out of jax.scipy.ndimage core into a separate package (e.g. dm-pix). No action needed now; re-check before implementing CROP in case it lands first. Source: spine Deferred (added by web-verify reviewer lens).
